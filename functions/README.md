# Leak Lookout Backend

Бэкенд для сервиса Leak Lookout, который сканирует репозитории GitHub и GitLab на наличие утечек API ключей и других чувствительных данных.

## Архитектура

Бэкенд Leak Lookout построен на основе Firebase Cloud Functions и использует следующие компоненты:

- **Firebase Firestore** - для хранения данных о сканированиях и обнаруженных утечках
- **Cloud Functions** - для обработки запросов и выполнения сканирований
- **Cloud Tasks** - для асинхронной обработки и масштабирования
- **GitHub API** - для доступа к репозиториям GitHub
- **GitLab API** - для доступа к проектам GitLab

## Основные компоненты

### Сканеры

- `enhancedScanner.ts` - улучшенный сканер с параллельной обработкой и оптимизированной логикой
- `leakDetector.ts` - детектор утечек с двухэтапным анализом для снижения количества ложных срабатываний
- `fileFilters.ts` - фильтры для исключения нерелевантных файлов

### Очереди задач

- `taskQueue.ts` - утилиты для работы с очередями задач
- `queueSetup.ts` - настройка и управление очередями задач

### Клиенты API

- `githubClient.ts` - клиент для работы с GitHub API
- `gitlabClient.ts` - клиент для работы с GitLab API

### Утилиты

- `batchProcessor.ts` - утилиты для параллельной обработки с ограничением нагрузки
- `utils.ts` - общие утилиты
- `types.ts` - типы данных
- `firestoreService.ts` - сервис для работы с Firestore
- `configService.ts` - сервис для работы с конфигурацией

## Улучшения

### 1. Параллельная обработка

Реализована параллельная обработка файлов и репозиториев с использованием пакетной обработки и Cloud Tasks:

- Обработка файлов пакетами с ограничением параллельности
- Асинхронная обработка репозиториев через очереди задач
- Экспоненциальная задержка при повторных попытках

Реализован двухэтапный анализ для более точного обнаружения утечек:

- Первичный анализ по шаблонам API ключей
- Вторичный анализ на основе энтропии
- Фильтрация известных ложных срабатываний
- Анализ контекста для снижения количества ложных срабатываний

### 3. Оптимизация фильтрации файлов

Добавлены фильтры для исключения нерелевантных файлов:

- Фильтрация по расширению файла
- Фильтрация по пути файла
- Фильтрация по размеру файла
- Фильтрация по контексту файла

### 4. Масштабируемая архитектура с очередями задач

Реализована архитектура на основе очередей задач:

- Очередь для сканирования репозиториев
- Очередь для обработки файлов
- Очередь для валидации утечек
- Настройка ограничений скорости и повторных попыток

### 5. Улучшенная обработка ошибок

Добавлена более гранулярная обработка ошибок:

- Обработка ошибок аутентификации
- Обработка ошибок ограничения скорости API
- Обработка сетевых ошибок
- Логирование с контекстом

## Настройка и запуск

### Предварительные требования

- Node.js 20 или выше
- Firebase CLI
- Доступ к проекту Firebase
- API ключи для GitHub и GitLab

### Установка зависимостей

```bash
cd functions
npm install
```

### Настройка переменных окружения

Создайте файл `.env` в директории `functions` со следующими переменными:

```
GITHUB_API_KEY_SECRET_NAME=GITHUB_API_KEY
GITLAB_API_KEY_SECRET_NAME=GITLAB_API_KEY
CLOUD_TASKS_LOCATION=us-central1
SCAN_QUEUE_NAME=scan-queue
```

### Локальный запуск

```bash
firebase emulators:start
```

### Деплой

```bash
firebase deploy --only functions
```

## Использование

### Запуск сканирования вручную

Для запуска сканирования вручную, создайте документ в коллекции `github_repos_to_scan` с следующими полями:

```json
{
  "repoName": "owner/repo",
  "provider": "github"
}
```

### Запланированное сканирование

Сканирование запускается автоматически каждые 24 часа с помощью Cloud Scheduler.

## Мониторинг и отладка

### Логи

Логи доступны в Firebase Console в разделе Functions.

### Статистика очередей

Для получения статистики очередей используйте функцию `getQueueStats` из модуля `queueSetup.ts`.
